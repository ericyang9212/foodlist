# 更新紀錄

重要改動的摘要（完整歷史見 `git log`）。

## 2026-07-21

### 修正：新增／編輯表單儲存失敗會關閉、內容全丟
- 根因：`AddEditPage` 呼叫 `onSave` 後不等結果就關閉，App 層也無條件關表單；網路一失敗，樂觀更新回滾＋表單已關，打到一半的筆記全部消失
- 改成 `onSave` 回傳成功與否：失敗時表單留在原地（內容不丟）直接重試，成功才由 App 層關閉；照片已上傳成功的話重試不會重傳第二份
- 兩處 `alert()`（新增頁圖片上傳失敗、靈感匣上傳失敗）改用全站統一的 toast

### 無障礙與互動品質（以 impeccable 稽核標準全站掃描）
- 對比拉到 WCAG AA：`#5d574c`（2.78:1）與 `#6d6557`（3.47:1）統一成 `#837b6e`（4.8:1）；眉標透明度 0.55/0.6 → 0.8/0.82
- 漸層字（`background-clip: text`）改實色 `#e0c68c`：原寫法在強制對比模式下標題會整段消失
- 圖示鈕點擊區補到 44×44（`::after` 撐熱區，版面不動）；補 18 個 `aria-label`
- 全域 `prefers-reduced-motion` 支援（先前只有跑馬燈處理）；表單控件納入鍵盤焦點金環；`transition: all` 改明確屬性（commit `86fa382`）

### 新增 `CLAUDE.md` 維護守則
- 行為守則（宣告與行動一致、範圍紀律、驗證＝執行）＋固定流程（編輯前重讀、驗證三連、記 CHANGELOG）＋專案關鍵事實（樂觀更新模式、照片存於靈感、店家型判別規則）＋設計系統底線；每次開新對話自動載入

## 2026-07-18

### 足跡頁「記一筆」：清單沒有的店也能直接記
- 足跡頁右上新增「＋ 記一筆」按鈕：填名稱、縣市、Google 地圖連結、吃的日期（預設今天、可補記）、照片、一句話感想
- 儲存時一併建立「嘗過」項目＋足跡紀錄，回訪抽籤、店家視角、足跡地圖立即生效；地圖連結內含座標會順手解析
- 任一步失敗（照片上傳／食物寫入／足跡寫入）會回滾前面已建立的資料，不留半套；sheet 留在原地可重試（commit `7c94476`）

### 修正：足跡卡片看不到照片
- 根因：照片常加在「食物」上（新增頁的截圖欄，存於靈感相簿），但足跡卡片只認記錄當下上傳的那張，看起來像照片消失
- 足跡沒有自己的照片時，退回顯示該食物的照片；既有足跡（如 7/17、7/18 那兩筆）立即補上，不用重傳（同 commit `7c94476`）

### 質感打磨：同風格提升整體 UI
- 冷灰全面暖化：全 app 一百多處純冷灰（`#0a0a0a`／`#161616`／`#1f1f1f`／`#2a2a2a`／`#555`／`#666`／`#777`…）整批遷移到帶香檳色偏的暖中性階，部分併回既有暖色 token；金色系不動。底色、邊線、灰字與香檳金融為同一世界
- 背景質地：`#root` 在頂部金暈外加極細顆粒（SVG 噪點 ~4%）與底部微弱餘燼光，暗部不再死平
- 細節工藝：原生控件走暗色系（`color-scheme: dark`）＋香檳 `accent-color`、輸入游標金色、日期選擇器圖示調暖、鍵盤焦點香檳細環、標題斷行 `text-wrap: balance`
- 六處空狀態「— —」統一升級為細金線＋✦ 的共用飾紋（`EmptyMark`）（commit `290fde4`）

### 修復 iOS Safari 自動縮放

- 全站 18 個 input/textarea/select 字級統一改 `text-base`（16px）——iOS Safari 在 input 字級 < 16px 時會強制 zoom in
- `src/index.css` 加保底規則 `input, textarea, select { font-size: 16px; }`（commit `99d7920`）
